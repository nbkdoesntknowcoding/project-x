# Very long paragraph

The Live Context Engine is a writing surface for AI workflows that lets teams capture, version, and publish prompts, instructions, and reference material in a single place that any MCP-aware client can consume. Writers compose in a Notion-class editor with collaborative cursors, real-time sync, slash commands, embedded math, and Mermaid diagrams; everything they type is normalised through the same Milkdown transformer on both client and server, persisted as both a Yjs CRDT state and canonical markdown, and served back to claude.ai through an OAuth-protected MCP endpoint that respects per-tenant access control. Behind the surface, every save runs in a single Postgres transaction that updates the Yjs state, the markdown column, and a content hash atomically; a content-hash short-circuit avoids re-embedding work when a Yjs update produced no semantic change. Snapshots into a doc-versions table fire every 50 store events as recovery anchors, and a SIGTERM drain forces a final write before the collab process exits during a rolling deploy. Multi-tenancy is enforced by Postgres row-level security policies on every table that holds tenant data, with a SET LOCAL app.tenant_id GUC inside every transaction that touches those tables, and a CI-enforced regression test that mints a JWT for the wrong tenant and asserts zero rows returned across SELECT, UPDATE, and DELETE paths. The browser side stays Astro 5 with React 19 islands, Tailwind v4 tokens for the design system, and Crepe as the editor wrapper around Milkdown; the server side is Fastify on Node 22 plus a separate Hocuspocus process on a dedicated port, both managed by the same pnpm dev command via concurrently in development and by Fly machines per process in production.
